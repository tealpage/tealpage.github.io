Code Sprints to Version 0.2
20/11/2022

With the design of the webpage now complete (in a single setting), all that’s “left” is to actually implement them in code. Whilst this might seem daunting at a glance, it wasn’t as bad as with the concept sketches from before, I have a decent idea of what needs to be done and roughly how to get them done. Trying to allow myself to build momentum by following the agile approach, I split up the tasks into self-contained sections or sprints of work which think I can tackle each in single sittings of roughly 4 to 5 hours.

Sprint I

The first of these was to complete the navigation bar for the webpage. I decided to go with this first as I knew I can capitalise on the progress from the first “failed” attempt, by just making minor adjustments to the existing colour schemes and a little additional code. Knowing this is probably fairly simple, I also wanted to add an “sticky” feature which would allow the navigation bar to stay at the top of the screen when scrolling.

For this I consulted W3School which was very handy with many examples code for various features (adaptations for many of these would end up in final version of the site). That being said, although the code for the nav-bar is largely identical their template exmaples, this still took a surprisingly long time… (this a reoccurring theme btw : D ). but I don't think this is a bad thing as I wanted to understand the code at least on a basic level and be familiar with various syntaxes (having not used HTML, CSS nor JavaScript before), which I think paid off in the long term as you will find out!

Working on the NavBar and adding in the assests

Sprint II

Following on from the day before and feeling fresh and motivated, I moved on to making a way to display the individual projects and their sub entry. In the original tutorial on Youtube a grid format was used, but I wanted something different to display clearly different project "threads".

For this drop down effect I wanted, I adapted the “acordian” feature (again) form W3School. This was done by taking the "accordian" code and adding a simple sub-menu (which I'm getting quite familiar with at this point), and also using a custom grid box display instead of the default button so that can better implement the design on the concept sketches.

Original Accordian Demo on W3School

Final Accordian Feature

Again, this took an long time. Actually most of it was fairly quick, but it was the very minor details like using a custom drop down arrow that would change both direction and colour which took most of the time... again it is also where most of the learning is achieved, as I tried out my own code approahces so I didn't particularly mind.

On a side note, I needed a way to have multiple project titles, and entries with the same displayes, but just slight differences in the titles. It is moments like these that you wish thtat there is more of an inheritance feature for HTML classes. Whilst I'm sure there is probably a smart way (I'm guessing using JavaScript to automatically create these objects from a stored table) that I’m not aware of (inheritance definitely do in css and this is used later on), I've decied to leave this for future me to figure out, trying to focus on achiving a minimal viable product for now. I've tackled this problem for now using custom coding snippets, but is still far from ideal...

Sprint III

Finally ending the week, I focused on creating a template for individual blog entries and fixing a “bug” that I have found. I figured for the former, I can qickly adapt the existing nav-bar using CSS inheritance to show information about the entries and make a baisc user interface to allow people to browse between them. I did also considered using custom "arrow" shape for the "next" and "previous" buttons, but held off on it as I think it would take an exceeding long amount of time.

Adapting the NavBar for the entries

After that, The day then moved toward fixing the bug. It was an issue where the scrolling would cause the page to jump at the point of the sticky navbar feature is supposed kicks in. I think this was caused by some of the template code form W3School being non-funcitonal code itself. I had to spend some time trying to hobble together a solution using newly learnt JavaScript. This once again took a some time, but I Was quite happy with the final result and the slick end results

Finishing all the coding needed, I excitly and joyeously began to writing the actual content for all the pages; naïvely typing away, unaware of how much of a task this is going to be just yet...

Sprint IV

You can entirely thank procrastination and sheer boredom for the current Contact and Home Pages (As you can see, they are entirely different) between the concept sketches and their current forms. Just as I naïvely thought last time that I was basically finish and just have to write out all the content in a day or two, it soon turned into a multi-day grind. I forgot that it can be a challenge to write pages and pages after barely reading and writing much doing engineering (we just grunt at each other instead in engineering I guess), turning to a bit of procrastination I then made the current contact and home pages.

Final Takeway From Coding up Version 0.2

I think building this website really validated this agile approach to completing and learning from projects. Having these sections that I can finish within a sitting, meant that I can alway look back on and be satisfied from my progrss, and hence usually feel fairly motivated.

Although this is not without its share of challenges, for example I definitely felt the lure to "mission creep", and incorporating some of the new features definitly always took a longer amount of time than expected. Also I felt much less motivated when I felt uncertain on how to approach a problem or lumping too much work in a single sprint. I guess these are the thing that I'll need to look out for but I'm definitely excited to try this method on more things in the future.

Finally in term of the website itself, I think there is still many improvements that I can make in the future. I would like to add features like contact forms, and I have definitely accuminated enough "technical debt" that probably warrent a rewrite of the entire website in a more profession way in the futre, but I guess this is good enough for now and I'm happy with the results seeing the progress made over a few weeks, starting from not knowing anything about HTML and CSS at all!
